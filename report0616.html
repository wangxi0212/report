MSCS分单优化 - 项目背景

业务痛点
员工离职后，其名下工单无人处理，导致流程停滞
需人工手动查询离职状态并调整处理人
分单规则计算后无法自动识别离职人员
影响业务流转效率与客户体验
核心成员

王西 需求分析
Project 01

MSCS分单优化 - 实施方案

1
用户表新增字段
MSCS用户表新增【是否离职】及【流程交接人】字段，存储OA离职用户信息
2
OA离职信息分发
OA离职审批流程结束后，通过RabbitMQ分发消息，MSCS消费并更新用户离职状态及交接人
3
处理人计算逻辑优化
计算得出处理人后，匹配用户表离职标识，若已离职则自动返回交接人工号

Project 02

遗留单责任申诉 - 项目背景

业务痛点
申诉订单审核完成后，需管理员手工做加减收调整费用
操作复杂，处理时间长，影响效率
财务报表无法自动对接，需手工调整
人工统计加工工作量大，易出错
项目目标
实现财务报表自动取数
减少人工统计加工，提升财务数据准确性与实时性
Project 02

遗留单责任申诉 - 实施方案

字段新增
VFDP申诉流程部件明细行增加【价格】、【板件ID】字段，获取原遗留单价格
价格审核栏
新增【成本收入分类】、【加收收入分类】、【计价收入分类】，支持自动联动计算
自动对接
申诉完成后自动对接结算中心扣款，无需人工干预
Project 02

遗留单责任申诉 - 收入分类逻辑

加收收入分类
商场责/运损责申诉公司责 → 计价收入分类的负值
公司责申诉商场责 → 计价收入分类
公司责责任二级申诉 → 默认为0，可调整
成本收入分类
商场责/运损责申诉公司责 → 计价收入分类
公司责申诉商场责 → 计价收入分类的负值
公司责责任二级申诉 → 默认为0，可调整
智能判断
若【加收收入分类】和【成本收入分类】均为0，则无需对接结算中心
